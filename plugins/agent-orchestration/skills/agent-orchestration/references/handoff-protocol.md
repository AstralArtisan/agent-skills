# 交接协议：原生记忆文件、台账、规格、handoff

本方法靠**文档**在代理之间传递状态，而不是靠聊天上下文。本文定义那几类文档各自的角色、字段，以及一条不可破的交接铁律。

## 交接铁律

子代理是**全新隔离上下文**，它读不到你和用户的对话，也不会自动读到任何文件。所以代理之间的交接，永远是这三步：

1. 你在派发 prompt 里**写明它要读的文件路径**；
2. 子代理**自己去读**这些文件；
3. 子代理把结果写进指定的输出文件，并在**末条消息**里回传摘要（改了哪些文件、写了哪个 handoff、跑了什么命令、结果、未解决项）。

推论：凡是子代理需要知道的事实，要么写进它被指路去读的文件，要么直接写进 prompt。"它应该能猜到"在这里不成立。

## 原生记忆文件：你的复位锚

宿主会把**项目级原生记忆文件**在每次会话 / 压缩后自动加载进**你（主 agent）**的上下文——但**不会**自动加载进子代理。所以它的角色是**你写给未来自己的复位锚**，不是给子代理看的。

- 映射：Claude Code → 项目根 `CLAUDE.md`；Codex → `AGENTS.md`；其他宿主 → 该宿主自动加载的项目级记忆文件。
- 只写**项目级**，**不要写用户全局的 `~/.claude/CLAUDE.md`**，以免污染全局配置。
- 里面放**锚定块**：你的身份与操作契约、任务一句话目标、流水线概览、指向 `ORCHESTRATION.md` 与 `TASK-SPEC.md` 的路径、当前阶段。**不要**把台账明细或规格全文堆进来——那些各有其文件，这里只放"够你复位"的最小信息 + 指针。模板见 `assets/MEMORY-ANCHOR.template.md`。
- 因为子代理读不到它，凡子代理需要的内容仍要在派发 prompt 里给路径。

## `ORCHESTRATION.md`：调度台账

项目管理台账，记录"谁在做什么、到哪了"。建议字段（完整模板见 `assets/ORCHESTRATION.template.md`）：

- **Environment**：工作目录、运行时 / 解释器路径、关键命令、前序产物路径、日期。
- **Agent Registry**：每个被拉起的子代理——ID、名字、角色、范围、输入、产出、状态、备注。
- **Pipeline & Workflow Status**：本任务专属流水线的各阶段 / 单元——属主代理、状态、对应 handoff / 证据、下一步。
- **Decisions**：开工后遇歧义时的自主决策与理由（让用户事后能看懂你为什么这么选）。
- **Known Risks**：已知风险与待观察项。
- **Git Commits**：每个验收门后的提交记录。

规则：拉起子代理就登记；代理完成就更新产出与状态；验收失败就记下失败原因与返工代理。收尾时，这份台账应能让用户读懂"整个项目是怎么被调度完成的"。

## `TASK-SPEC.md`：权威规格

本任务"做什么、怎样算成"的唯一权威来源（完整模板见 `assets/TASK-SPEC.template.md`）。建议字段：

- 交付物清单
- 完成口径 / 验收口径（具体命令、期望输出、指标）
- 约束（技术栈、风格、时限、禁区）
- 前序依赖与产物路径
- 运行与测试环境
- 阶段划分（与台账的 Pipeline 呼应）
- 未决项 / 占位符（只有用户本人能补的事项）

实现代理和验收代理都以本文件为准；它和台账的区别是：**规格是"要求"，台账是"进度"**。

## handoff 文件：单元之间的接力棒

`handoff/` 目录放代理之间的交接文档。文件名、数量、阶段**都不写死**，按你的流水线裁剪。常见内容（编号模板见 `assets/handoff/`）：

- 勘查类（如 `00_context-baseline.md`）：环境与前序产物的事实。
- 需求类（如 `01_requirements.md`）：从原始资料提炼的需求摘录。
- 实现交接（如 `02_impl-handoff.md`）：改了哪些文件、实现了什么、怎么运行、怎么测、已知限制。
- 验收交接（如 `03_validation.md`）：验收跑了哪些命令、真实输出证据、发现的缺陷、是否通过。
- 最终验收（如 `04_final-acceptance.md`）：通盘核对的硬结论。
- 空白通用模板（`_handoff.template.md`）：要新增阶段时复制裁剪。

每份 handoff 都应当**自包含到下游凭它 + 规格就能接着干**，不需要回头猜上一个代理的内心活动。

## 三类文档怎么分工（一句话）

- **原生记忆文件**：给"未来的你"复位用——最小操作契约 + 指针。
- **台账（ORCHESTRATION.md）**：进度与决策的流水记录。
- **规格（TASK-SPEC.md）**：不变的要求与验收口径。
- **handoff/**：一个单元交给下一个单元的接力棒。
